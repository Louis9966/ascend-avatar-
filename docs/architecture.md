# ascend-avatar 技术架构

## 目标

在单卡昇腾 910B（`/dev/davinci7`）容器内实现实时对话 2D 数字人：

用户文本 → 流式 LLM → 子句切分 → 流式 TTS → MuseTalk 唇形同步 → 视频片段 → Gradio 前端。

## 技术选型

| 模块 | 选型 | 说明 |
|------|------|------|
| LLM | 本地 MindIE 服务 | OpenAI 兼容接口 `http://192.168.1.117:1025/v1`，模型 `qwen3_32b` |
| TTS | edge-tts | 微软 Edge TTS 在线服务，支持中英多音色，首包约 800ms |
| THG | MuseTalk v1.5 | 音频驱动唇形，基于预计算 avatar 素材包运行 |
| 前端 | Gradio 4.44.1 | 输入框、音色/语言选择、字幕、视频播放器、延迟指标 |
| 运行环境 | `ascend-pytorch:24.0.0-A2-2.1.0-ubuntu20.04` 容器 | 仅操作容器，不修改宿主机 |

## 数据流

```
+--------+   +----------------+   +-----------+   +----------+   +---------+
| 用户输入 | → | LLM 流式文本    | → | 子句切分   | → | edge-TTS | → | WAV音频 |
+--------+   +----------------+   +-----------+   +----------+   +---------+
                                                                     ↓
+----------+   +-----------+   +---------+                        +----------+
| 字幕展示 | ← | 句子视频片段 | ← | MuseTalk | ← 预计算 avatar ← | 音频特征  |
+----------+   +-----------+   +---------+                        +----------+
       ↓
+-----------+
| 合并 MP4  |
+-----------+
       ↓
+-----------+
| Gradio视频 |
+-----------+
```

## 模块说明

- `src/llm_client.py`：OpenAI 兼容流式客户端，记录首 token 延迟。
- `src/tts_engine.py`：edge-tts 封装，语言检测，音色选择。
- `src/thg_engine.py`：MuseTalk 模型加载、avatar 预处理、逐句推理生成视频帧。
- `src/pipeline.py`：串联 LLM/TTS/THG，维护对话历史，合并句子视频为连续 MP4。
- `src/webui.py`：Gradio 交互页面，异步生成器实时更新字幕与视频。

## 延迟预算（设计目标）

- LLM 首 token：~200–500ms
- TTS 首包/整句：~300–1000ms
- MuseTalk 首帧：依赖 NPU/CPU 推理速度，目标首视频 ≤1.5s
- 端到端中句（~20 字）：<5s

## 已知限制

- 当前实现按句子生成 MP4 片段并合并播放，未启用 RTMP/WebRTC 真流式传输。
- edge-tts 依赖外部网络；若网络受限需替换为本地 TTS（如 MeloTTS / PaddleSpeech）。
- MuseTalk 首次 avatar 预处理需要 MMLab 依赖，耗时数分钟。
